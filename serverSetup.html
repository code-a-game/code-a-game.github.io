Setting Up Your Own Server

Follow the guide below to set up your own server. These guidelines assume that you are already somewhat familiar with Git and Github.

  1. Fork jbakse/p5.party or code-a-game/p5.party repository on Github. See here for more information about forking repositories.
  2. Create a Heroku account and log in.
  3. From the main dashboard, select New 췉 Create New App
    1. Enter a name for your app in App name. You must choose your own unique name.
    2. Your account name should already be shown in the App owner field.
    3. Choose the region closest to you and your users.
    4. Click Create app.
  4. You should now be taken to the Deploy tab for your project.
    1. Select Github as the Deployment method
    2. If you belong to multiple github organizations, select the one you used to fork the repo.
    3. Type part of your fork name and press Search. You should see your fork in the resulting list.
    4. Click the Connect button next to your fork.
    5. In Manual deploy, verify that the master branch is selected, then click Deploy Branch.
    6. Heroku will pull the source from your branch and launch the server. This may take a few minutes.
    7. If all went well, you should now have your own server running on Heroku. Click Open app to open the server in your browser.
      • If the server startup succeeded, you will see "Endpoint not found".
      • If it failed, you will see "There's nothing here, yet".
  5. Point your p5.party sketch to the new server.
    • Use the domain of your server - something like yourserver.herokuapp.com - when you call partyConnect() in your sketch.

Congratulations! Hopefully your sketch now works with your new server!
